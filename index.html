☁️ AWS Execution Guide

Docker & IIS on AWS EC2

Step-by-step, no-theory execution guides. Follow once, works every time.

PART 1

Create Instance

1
Go to AWS → EC2
2
Click Launch Instance
3
Set the following:
  • Name → anything (e.g., docker-lab)
  • OS → Amazon Linux (or Ubuntu)
  • Instance type → t2.micro
  • Key pair → create / download
PART 2

Security Settings

In Network settings → Security group, add these inbound rules:

TypePortNote
SSH22Default
HTTP80⚠️ IMPORTANT
If you missed HTTP port 80: Security Groups → Edit inbound rules → Add HTTP (80)
PART 3

Launch & Connect

1
Click Launch Instance
2
Go to Instances and select your instance
3
Click Connect → EC2 Instance Connect → Connect
4
✅ Terminal opens in browser
PART 4

Install Docker (Amazon Linux)

bash
sudo dnf update -y
sudo dnf install docker -y

Start and enable Docker:

bash
sudo systemctl start docker
sudo systemctl enable docker

Give your user Docker permissions:

bash
sudo usermod -aG docker ec2-user
After running the above, type exit and reconnect for permissions to take effect.
PART 5

Create Project

bash
cd ~
mkdir docker-project
cd docker-project
PART 6

Create Webpage

bash
nano index.html

Paste this inside the file:

html
<h1>Docker Working on AWS 🚀</h1>
Save with: Ctrl + X → Y → Enter
PART 7

Run Docker

Pull the Apache image:

bash
docker pull httpd

Run the container:

bash
docker run -d -p 80:80 -v $(pwd):/usr/local/apache2/htdocs/ httpd
PART 8

Verify Container

bash
docker ps
You should see a running container listed in the output.
PART 9

Get Public IP

1
Go to EC2 → Instances
2
Copy the Public IPv4 address (e.g., 100.24.58.31)
PART 10

Open Website

Open your browser and go to:

url
http://<your-public-ip>
✅ Final Result:   Docker Working on AWS 🚀
BONUS

Container Management Commands

Remove a specific container and re-run:

bash
docker ps -a
docker rm -f <container_id>
docker run -d -p 80:80 -v $(pwd):/usr/local/apache2/htdocs/ httpd

Or remove ALL containers at once:

bash
docker ps -a
docker rm -f $(docker ps -aq)
⚠️

Common Errors & Fixes

❌ apt not found
Use dnf instead — this is Amazon Linux, not Ubuntu.
Replace apt install with dnf install
❌ Site not opening in browser
Check: Security group → Inbound rules → HTTP port 80 must be open (0.0.0.0/0)
❌ Docker permission denied
Run the following, then exit and reconnect:
bash
sudo usermod -aG docker ec2-user
exit
❌ Wrong IP used
Always use the Public IPv4 address.

1

2
Click Add Roles and Features
3
Click Next → Next → Next
4
Check ✅ Web Server (IIS) → Click Install
PART 6

cmd
cd C:\inetpub\wwwroot
PART 7

Delete Default Files

cmd
del *.*

cmd

TypePortSource

⚠️

Common Errors & Fixes

❌ Site not opening
Check all three causes:
  • Port 80 not open in Security Group
  • IIS not restarted — run iisreset
  • Wrong IP — use Public IPv4, not Private
❌ Page takes too long to respond
Wait 1–2 minutes after setup before trying to open the site.
BONUS

Verify via curl (from your PC)

Open Command Prompt on your laptop and run:

cmd
curl http://<your-public-ip>
Example output you should see: <h1> hello </h1> — confirms your IIS server is serving content correctly.
