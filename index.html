🚀 DOCKER ON AWS – FULL EXECUTION GUIDE

🔹 PART 1 – CREATE INSTANCE

Go to AWS → EC2 → Launch Instance

  • Name → docker-lab
  • OS → Amazon Linux / Ubuntu
  • Instance → t2.micro
  • Key pair → create/download

🔹 PART 2 – SECURITY SETTINGS

  • SSH → port 22
  • HTTP → port 80

🔹 PART 3 – LAUNCH & CONNECT

Use EC2 Instance Connect

🔹 PART 4 – INSTALL DOCKER

sudo dnf update -y
sudo dnf install docker -y
sudo systemctl start docker
sudo systemctl enable docker
sudo usermod -aG docker ec2-user
exit

🔹 PART 5 – CREATE PROJECT

cd ~
mkdir docker-project
cd docker-project

🔹 PART 6 – CREATE WEBPAGE

nano index.html

<h1>Docker Working on AWS 🚀</h1>

🔹 PART 7 – RUN DOCKER

docker pull httpd
docker run -d -p 80:80 -v $(pwd):/usr/local/apache2/htdocs/ httpd

🔹 PART 8 – VERIFY

docker ps

🔹 PART 9 – ACCESS WEBSITE

http://your-public-ip

⚠️ COMMON ERRORS

  • apt not found → use dnf
  • Port 80 not open
  • Wrong IP

🔹 EXTRA COMMANDS

docker ps -a
docker rm -f <container_id>

docker ps -a
docker rm -f $(docker ps -aq)
docker run -d -p 80:80 -v $(pwd):/usr/local/apache2/htdocs/ httpd

🚀 WEEK 5 (WINDOWS + IIS) — FULL EXECUTION GUIDE

This is your personal checklist (use anytime)

🔹 PART 1 – CREATE INSTANCE

  1. Go to AWS → EC2
  2. Click Launch Instance
  • Name → Week5-Server
  • OS → Microsoft Windows Server
  • Instance → t3.micro
  • Key pair → create/download

🔹 PART 2 – SECURITY GROUP

  • RDP → port 3389
  • HTTP → port 80

🔹 PART 3 – LAUNCH

  • Status = Running
  • Checks = 2/2 or 3/3

🔹 PART 4 – CONNECT (RDP)

  1. Select instance
  2. Click Connect
  3. Go to RDP Client tab
  1. Download RDP file
  2. Get Password
  3. Upload .pem
  4. Decrypt Password
  5. Copy password
Username: Administrator

🔹 PART 5 – INSTALL IIS

  1. Open Server Manager
  2. Add Roles and Features
  3. Select Web Server (IIS)

🔹 PART 6 – GO TO WEBSITE FOLDER

cd C:\inetpub\wwwroot

🔹 PART 7 – DELETE DEFAULT FILES

del *.*

🔹 PART 8 – CREATE YOUR PAGE

echo ^Week 5 IIS Server Working 🚀^ > index.html

🔹 PART 9 – RESTART IIS

iisreset

🔹 PART 10 – CHECK SECURITY GROUP

TypePortSource
HTTP800.0.0.0/0

🔹 PART 11 – GET IP

Go to EC2 → Instances → Copy Public IPv4

http://your-public-ip

🔹 FINAL OUTPUT

Week 5 IIS Server Working 🚀

⚠️ COMMON ERRORS

  • Port 80 not open
  • IIS not restarted
  • Wrong IP

🧠 MEMORY TRICK

EC2 → RDP → IIS → HTML → IP
